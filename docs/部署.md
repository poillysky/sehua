# sehuatang NAS 全栈部署

> 目标：**PostgreSQL + Backend + Admin + Search** 全部跑在家庭 NAS。  
> 镜像在 **GitHub Actions** 构建并推到 **GHCR**；NAS **只拉取、不编译**业务镜像。  
> 架构原则见 [架构.md](./架构.md)。

---

## 1. 总流程

```text
┌─────────────┐     push      ┌──────────────┐     build/push     ┌─────────┐
│ 开发机      │ ────────────► │ GitHub 仓库  │ ─────────────────► │  GHCR   │
│ 改代码/验证 │               │ + Actions    │                    │ 三镜像  │
└─────────────┘               └──────────────┘                    └────┬────┘
                                                                       │ pull
                                                                       ▼
                                                              ┌────────────────┐
                                                              │ 家庭 NAS       │
                                                              │ compose up -d  │
                                                              │ 四容器常驻     │
                                                              └────────────────┘
```

| 产物 | 镜像名（示例） |
|------|----------------|
| 后端 | `ghcr.io/<owner>/sehuatang-backend:latest` |
| 管理 | `ghcr.io/<owner>/sehuatang-admin:latest` |
| 搜索 | `ghcr.io/<owner>/sehuatang-search:latest` |
| 数据库 | `postgres:16-alpine`（官方，不自建） |

`<owner>` 替换为 GitHub 用户名或组织名。私有仓库镜像需在 NAS 上 `docker login ghcr.io`。

## 2. 容器拓扑

```text
                    ed2k-nas-network（bridge）
     ┌──────────────────────┬──────────────────────┬───────────────┐
     │                      │                      │               │
 postgres:5432         backend:8080           admin:80        search:3000
     │                      │                      │               │
     └──────── 宿主机端口 ──┴──────────────────────┴───────────────┘
              5433                 8080                 8081            3010
```

| 服务 | 容器名建议 | 宿主机端口 | 说明 |
|------|------------|------------|------|
| postgres | `sehuatang-postgres` | 5433 | 给本机工具连库；容器互访用主机名 `postgres` |
| backend | `sehuatang-backend` | 8080 | API + Playwright；`shm_size: 2gb` |
| admin | `sehuatang-admin` | 8081 | Nginx 静态；经代理访问 `/api` |
| search | `sehuatang-search` | 3010 | Next standalone |

内部连接：

- backend → `postgres:5432`
- search → `postgres://…@postgres:5432/ed2k`
- 浏览器 → `http://NAS_IP:3010`（搜）、`http://NAS_IP:8081`（管）

## 3. 推荐目录（NAS）

以飞牛 / 通用 Linux 为例（群晖把前缀换成 `/volume1/...`）：

```text
/vol1/1000/Docker/sehuatang/
├── docker-compose.yml      # 或从仓库拷贝 deploy/docker-compose.nas.yml
├── .env                    # 密码、代理；勿提交 Git
├── data/
│   ├── postgres/           # PG 数据
│   ├── backend/            # cookies、预览图、可选 auth.sqlite
│   ├── search/             # next-web：p115-config.json
│   └── search-cache/       # next-web：.next/cache（可丢）
└── update.sh               # 一键 pull + up
```

`.env` 最少项：

```bash
POSTGRES_USER=postgres
POSTGRES_PASSWORD=<强密码>
POSTGRES_DB=ed2k
# 论坛出网代理（按需）
# WEB_CRAWLER_PROXY=http://192.168.2.88:7893
GHCR_OWNER=<你的GitHub用户名>
INITIAL_ADMIN_USERNAME=admin
INITIAL_ADMIN_PASSWORD=<改掉默认密码>
```

## 4. Compose 约定（pull 模式）

业务服务**只写 `image:`**，不写 `build:`，确保 NAS 不编译。

要点：

```yaml
services:
  postgres:
    image: postgres:16-alpine
    ports: ["5433:5432"]
    volumes:
      - ./data/postgres:/var/lib/postgresql/data
      # 首次初始化：挂载 migrations，或改用 backend 启动时 migrate
    shm_size: 1g
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U $$POSTGRES_USER -d $$POSTGRES_DB"]

  backend:
    image: ghcr.io/${GHCR_OWNER}/sehuatang-backend:latest
    ports: ["8080:8080"]
    shm_size: 2g
    environment:
      POSTGRES_HOST: postgres
      POSTGRES_PORT: 5432
      WEB_CRAWLER_PROXY: ${WEB_CRAWLER_PROXY:-}
    volumes:
      - ./data/backend:/app/data
    depends_on:
      postgres:
        condition: service_healthy

  admin:
    image: ghcr.io/${GHCR_OWNER}/sehuatang-admin:latest
    ports: ["8081:80"]
    # Nginx 内将 /api /static /health 反代到 backend:8080（镜像内配置）

  search:
    image: ghcr.io/${GHCR_OWNER}/sehuatang-search:latest
    ports: ["3010:3000"]
    environment:
      POSTGRES_DB_URL: postgres://${POSTGRES_USER}:${POSTGRES_PASSWORD}@postgres:5432/${POSTGRES_DB}
    volumes:
      - ./data/search:/app/data              # 115 Cookie 等
      - ./data/search-cache:/app/.next/cache # 可选
    depends_on:
      postgres:
        condition: service_healthy
```

### 持久化对照

| 服务 | 卷 | 丢了会怎样 |
|------|-----|------------|
| postgres | `data/postgres` | **全库没了** |
| backend | `data/backend` | 论坛 Cookie、预览图没了；库中资源还在 |
| search | `data/search` | 115 Cookie/目录配置没了，要重填 |
| search | `data/search-cache` | 仅缓存，可忽略 |
| admin | （无） | 无状态 |

> 仓库文件：`deploy/docker-compose.nas.yml`、`.env.example`、`update.sh`。  
> `ed2k/` 仅本地参考，**不入 Git**。

### Schema 初始化

二选一（推荐 B）：

| 方式 | 做法 |
|------|------|
| A | 空库首次启动时把 `database/migrations/*.sql` 挂进 postgres `initdb.d`（仅**全新库**有效） |
| B | backend 启动 lifespan 跑 `ensure_ed2k_schema()` / migrate（与现网一致，**推荐**） |

已有 `ed2k` / `tang98` 数据卷时：**不要**再跑会冲突的 init 脚本，沿用 backend 迁移即可。

## 5. 镜像构建与推送（CI）

触发：`main` 分支 push，或手动 `workflow_dispatch`。

```text
.github/workflows/docker.yml
  ├─ build backend  → ghcr.io/<owner>/sehuatang-backend:<sha> + :latest
  ├─ build admin    → …/sehuatang-admin:…
  └─ build search   → …/sehuatang-search:…
```

| 注意 | 说明 |
|------|------|
| 仓库私有 | Package 默认跟随仓库可见性；NAS 用 PAT（`read:packages`）登录 |
| 多架构 | 优先 `linux/amd64`；ARM NAS 再补 `linux/arm64`（Playwright 要额外验证） |
| 构建缓存 | Actions cache；backend 层最重（Chromium） |
| 标签 | 生产可钉 `v1.2.3`，日常用 `latest` |

本地试构建（开发机有 Docker 时）：

```bash
docker build -t sehuatang-backend ./backend
docker build -t sehuatang-admin ./frontend/admin
docker build -t sehuatang-search ./next-web
```

## 6. NAS 一键拉取 / 更新

首次：

```bash
docker login ghcr.io -u <GitHub用户名> -p <PAT>
cd /vol1/1000/Docker/sehuatang
docker compose pull
docker compose up -d
docker compose ps
curl -s http://127.0.0.1:8080/health
```

之后更新（`update.sh`）：

```bash
#!/bin/sh
set -e
cd "$(dirname "$0")"
docker compose pull
docker compose up -d
docker image prune -f
echo "OK $(date)"
```

健康检查清单：

| URL | 期望 |
|-----|------|
| `http://NAS_IP:8080/health` | backend OK |
| `http://NAS_IP:8081` | 管理登录页 |
| `http://NAS_IP:3010` | 搜索首页 |
| `http://NAS_IP:5433` | （可选）PG 外部工具 |

默认管理账号首次启动后**立刻改密**（`INITIAL_ADMIN_*` 勿用文档默认值长期放公网）。

## 7. 网络与代理

| 场景 | 建议 |
|------|------|
| 仅家庭局域网访问 | 不暴露 8080/8081/3010 到公网；走 NAS 防火墙 |
| 爬虫访问论坛失败 | 设置 `WEB_CRAWLER_PROXY` 指向局域网代理 |
| 管理 Cookie 跨端口麻烦 | 上 Caddy/Nginx：同域 `/admin` + `/api` |
| 搜索要 HTTPS | 反代 3010，证书用局域网 CA 或 Tailscale |

同域反代示例（逻辑）：

```text
https://nas.local/admin/     → admin:80
https://nas.local/api/       → backend:8080
https://nas.local/static/    → backend:8080
https://nas.local/           → search:3000   # 或独立子域 search / admin
```

## 8. 资源与稳定性

| 项 | 建议 |
|----|------|
| 内存 | Docker 可用内存 **≥ 8GB**；同时开爬 + 搜峰值留余量 |
| shm | backend **2g**；postgres **1g** |
| CPU | x86 NAS 优先；群晖 ARM 需自测 Chromium |
| 重启策略 | `restart: unless-stopped` 或 `always` |
| 爬虫节奏 | 控制并发与间隔，避免把 NAS 打满、避免触发站点风控 |
| 备份 | 定期停写或 `pg_dump`；备份 `data/postgres` + `data/backend` |

Playwright 在 NAS 上挂掉时优先查：shm 过小、内存 OOM、缺代理、论坛 Cookie 过期。

## 9. 开发 vs 生产

| | 开发机（`start.bat`） | NAS 生产 |
|--|----------------------|----------|
| backend | uvicorn `--reload` :8080 | 容器镜像 |
| admin | Vite :8081 | Nginx 静态 :8081 |
| search | `next dev` :3010 | standalone :3010 |
| DB | 可连 NAS 已有 PG，或本地 | compose 内 postgres |
| 更新 | 改代码热重载 | `update.sh` pull |

开发不必在 NAS 上挂载源码；**以镜像交付**为准。

### 首次推送到 GitHub

1. 安装 [Git for Windows](https://git-scm.com/download/win)，新开终端确认 `git --version`。  
2. 浏览器登录 GitHub → **New repository**（建议 Private，空仓库不要勾 README）。  
3. 在本机项目根目录：

```powershell
cd Z:\project\sehuatang
git init
git add .
git commit -m "Initial commit: NAS deploy + Docker images"
git branch -M main
git remote add origin https://github.com/<你的用户名>/sehuatang.git
git push -u origin main
```

4. 打开仓库 **Actions**，确认镜像构建成功；Packages 出现 `sehuatang-backend` / `admin` / `search`。  
5. NAS 上按 §6 拉取部署。
